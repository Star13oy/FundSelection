# ADR-001 总体架构（MVP）

- 状态：Accepted
- 日期：2026-03-28

## 背景
产品目标是为理财初学者提供 A 股基金量化选基工具，要求：
- 文档先行
- 评审门禁
- 推荐可解释
- 单元测试覆盖率 >= 90%
- 集成测试 100% 通过

## 决策
采用前后端分离架构：
- 前端：React + Vite + TypeScript
- 后端：Python + FastAPI
- 数据层：MVP 使用内置样例数据（后续可切 PostgreSQL）
- 测试：
  - 后端：pytest + pytest-cov
  - 前端：Vitest + Testing Library
  - 集成：pytest（API 端到端链路）

## 关键模块
1. Fund Screening（筛选）
2. Scoring Engine（基础因子 + Policy Overlay）
3. Explainability（加分项/扣分项/风险提示）
4. Comparison（2-5 只基金对比）
5. Watchlist（观察池）

## 约束
- Policy 因子为 Overlay，不替代基础因子。
- Overlay 权重按基金类型和风险偏好动态变化，且上限 25%。
- 禁止未来函数：政策事件仅在发布日期后生效。

## 门禁
- Gate-DOC：PRD + ADR + API + TEST STRATEGY 评审通过后方可开发。
- Gate-CODE：lint/type-check/unit/integration 全通过后方可合并。
- 前端 unit 门槛：`lines/statements >=90`，`functions/branches >=80`（以 `frontend/vite.config.ts` 为准）。
